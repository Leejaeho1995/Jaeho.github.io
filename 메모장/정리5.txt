개발 환경 구축
- 개발 프로젝트를 이해하고 소프트웨어 및 하드웨어 장비를 구축하는 것. (도커)

-응용 소프트웨어가 운영될 환경과 유사한 구조로 구축

-분석 단계의 산출물을 바탕으로 하드웨어,소프트웨어를 선정


------------------------------------------------------------------------

하드웨어 환경 : 사용자와의 인터페이스 역할을 하는 client, 
client와 통신하여 서비를 제공하는 sever로 구성된다.
클라이언트 종류 :개인용PC 또는 스마트폰

client (사용자)웹브라우저 <-------->웹 어플리케이션 서버(WAS) <--------->DB서버 <----->파일서버

 ⓐ 서버의 종류 와 특징
웹서버 (Web Server)     :1. 클라이언트로부터 직접 요청을 받아 처리함.  2. 저용량의 정적 파일들을 제공함.

웹 에플리케이션서버 (WAS:Wen Application Server)    :1. 동적 서비스를 제공. 2.웹서버와 데이터베이스 서버
또는 웹서버와 파일서버 사이에서 인터페이스 역할을 수행

데이터베이스 서버 (DB Server)    : 1. 데이터베이스와 이를 관리하는 DBMS를 운영함.

파일 서버 ( File Server)    : 1. 데이터베이스에 저장하기에 비효율적이거나, 서비스 제공을 목적으로 유지하는 파일들을 저장함.

----------------------------------------------------------------------------

소프트웨어 환경 : 
클라이언트 , 서버운영을 위한 소프트웨어 : 시스템 소프트웨어

개발에 사용되는 소프트웨어 : 개발 소프트웨어

시스템 소프트웨어의 종류 : 운영체제(OS), 웹서버 ,WAS운용을 위한 서버 프로그램, DBMS..등

개발 소프트웨어의 종류 
요구사항관리 도구 : 요구사항의 수집,분석,추적등을 편리하게 도와주는 소프트웨어 (기획자)
설계/모델링 도구: UML(Unified Moedling Language.통합 모델링 언어)을 지원하며, 개발의 전 과정에서 설계 및 모델링을 도와주는 소프트웨어
구현도구 : 개발언어를 통해 애플리케이션의 실제 구현을 지원하는 소프트웨어
빌드도구 : 구현도구를 통해 작성된 소스의 빌드 및 배포, 라이브러리 관리를 지원하는 소프트웨어
테스트도구 : 모듈들이 요구사항에 적합하게 구현되었는지 테스트하는 소프트웨어
형상 관리도구 : 산출물들을 버전별로 관리하여 품질 향상을 지원하는 소프트웨어 ( github )
--------------------
웹 서버의 기능 (ⓐ)
HTTP/HTTPS지원 : 브라우저로부터 요청을 받아(request) 응답(response)할 때 사용되는 프로토콜
(Hyper Text Transfer Protocol)
통신기록 (Communication Log) : 처리한 요청들을 로그 파일로 기록하는 기능
정적 파일 관리 (Managing Static Files) : HTML . CSS 이미지 등의 정적 파일들을 저장하고 관리하는 기능
대역폭의 제한 (Band Width Throtting) :네트워크 트래픽의 포화를 방지하기 위해 응답 속도를 제한하는 기능
가상 호스팅 (Virual Hosting) : 하나의 서버로 여러 개의 도메인 이름을 연결하는 기능
인증 (Authentication) : 사용자가 합법적인 사용자인지를 확인하는 기능 (다른웹사이트 로그인기능 ex 카카오 네이버)

--------------------

개발언어를 선정기준
적정성 - 소프트웨어의 목적에 적합해야 한다.
효율성 - 코드의 작성 및 구현이 효율적 이어야 한다.
이식성 - 다양한 시스템 및 환경에 적용이 가능하여야 한다.
친밀성 - 개발 언어에 대한 개발자들의 이해도와 활용도가 높아야 한다.
범용성 - 다른 개발 사례가 존재하고 여러분야에서 활용되고 있어야 한다.
